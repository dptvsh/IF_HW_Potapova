## Требования к окружению

- **Java**: 21 или выше (проект собирался с target 21). Рекомендуется версия 25.0.2, на которой проводилось тестирование.
- **Maven**: 3.9.14 или новее.

## Установка и запуск тестов

1. **Клонировать ветку с тестами**
   ```bash
   git clone -b HWF_API https://github.com/dptvsh/IF_HW_Potapova.git
   cd IF_HW_Potapova
   ```
2. **Запустить все тесты**
    ```bash
   mvn clean test
    ```

## Запуск отдельных тестов

```bash
    mvn test -Dcucumber.filter.tags=@Auth
```

Доступные тэги:
- @Auth – общий тег для всех сценариев, связанных с аутентификацией (регистрация, вход, выход). Позволяет запускать только тесты авторизации.
- @RickMorty – тег функции для сценария проверки данных о персонажах Rick and Morty.
- @Positive – тег для позитивных (успешных) сценариев.
- @Negative – тег, обозначающий негативные тесты (ошибочные действия, ожидание ошибок).
- @Register – тег функции для сценариев регистрации.
- @Login – тег функции для сценариев входа (логина).
- @Logout – тег функции для сценариев выхода.
- @SuccessRegister – тег конкретного сценария успешной регистрации. Относится к позитивной проверке.
- @SuccessLogin – тег сценария “Успешный вход”.
- @SuccessLogout – тег сценария “Успешный выход”.
- @LoginWrongUsername – тег сценария “Вход с неверным username” (негативный).
- @LoginWrongPassword – тег сценария “Вход с неверным паролем” (негативный).
- @LogoutInvalidToken – тег сценария “Попытка выхода с невалидным токеном” (негативный)

## Генерация и просмотр Allure-отчёта

После прогона тестов:

```bash
   # Сгенерировать отчёт из результатов
   mvn allure:report

   # Открыть отчёт в браузере
   mvn allure:serve
```

## Запуск тестов и отчётов через Maven Tool Window (IntelliJ IDEA)

Если вы работаете в IntelliJ IDEA, можно не использовать командную строку, а управлять Maven-фазами через графическое меню.

### Запуск тестов:
1. Разверните **Lifecycle**
2. Дважды кликните на `clean`
   → IDEA запустит `mvn clean`, в результате будут удалены старые артефакты.
3. Дважды кликните на `test`  
   → IDEA запустит `mvn test`, результаты появятся во встроенной консоли.

### Формирование Allure-отчёта:
1. Разверните **Plugins → allure**
2. Дважды кликните на **`allure:report`**  
   → после выполнения отчёт будет сгенерирован в `target/site/allure-maven-plugin/index.html`.
3. Чтобы открыть отчёт сразу в браузере, выполните **`allure:serve`** из того же меню.
